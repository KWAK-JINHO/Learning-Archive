# 동기 vs 비동기

핵심은 작업의 완료 여부를 누가 신경 쓰는가?  
호출된 함수의 작업 완료를 호출자가 직접 챙기면 동기, 아니면 비동기

## 동기(Synchronous)
호출자가 호출된 함수의 완료를 직접 확인하고 기다림.
- 흐름: 작업의 순서가 보장되며, 결과가 반환될 때까지 다음 작업을 수행하지 않거나 완료 여부를 계속 체크함.

## 비동기(Asynchronous)
호출자가 호출된 함수의 완료 여부를 직접 확인하지 않음.
- 흐름: 호출 시 콜백(Callback)이나 Future 객체를 넘겨주며, 호출자는 자신의 작업을 계속 수행함. 작업이 끝나면 통보를 받음.

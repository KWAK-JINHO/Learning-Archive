# OLTP vs OLAP

데이터베이스가 처리하는 작업의 성격(Workload)에 따라 시스템을 크게 두 가지 유형으로 분류한다.  
**블록 I/O 효율성**은 주로 OLTP 환경을 최적화하기 위한 기술들이다.

## 2. OLTP vs OLAP

| 구분         | OLTP (Online Transaction Processing) | OLAP (Online Analytical Processing) |
|:-----------|:-------------------------------------|:------------------------------------|
| **목적**     | 실시간 비즈니스 운영 및 트랜잭션 처리                | 의사결정 지원 및 데이터 분석                    |
| **데이터 단위** | 개별 행(Row) 중심                         | 대량의 집계 데이터(Batch)                   |
| **주요 작업**  | INSERT, UPDATE, DELETE (쓰기/읽기 혼합)    | SELECT (복잡한 읽기 중심)                  |
| **응답 속도**  | 밀리초(ms) 단위의 즉각적인 응답 필요               | 수 초 ~ 수 분 (대량 처리 위주)                |
| **인덱스 전략** | 특정 행을 찾기 위한 인덱스 활용 극대화               | 인덱스보다는 Full Scan이나 컬럼형 저장 방식 활용     |

## OLTP

- 소량의 블록을 빈번하게 읽고 쓴다.
- **Random I/O** 최소화와 캐시 적중률(Hit Ratio)이 성능의 핵심이다.
- 데이터 무결성을 위해 정규화를 철저히 하여 중복을 제거한다.
- 은행 계좌 이체, 쇼핑몰 주문 프로세스 등..

## OLAP

- 수만 개의 블록을 한꺼번에 읽는다.
- **Sequential I/O**를 활용하여 한 번에 많은 블록을 가져오는 것이 유리하다.
- Join 연산 비용을 줄이기 위해 일부러 데이터를 중복시키는 비정규화나 스타 스키마 구조를 사용한다.
- 전년 대비 매출 성장률 분석, 사용자 연령대별 선호 루틴 통계 등..

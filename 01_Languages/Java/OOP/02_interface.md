# 인터페이스와 캡슐화

## 인터페이스란 무엇인가

객체가 외부와 상호작용하기 위해 제공하는 공개된 행동

---

## 객체는 인터페이스로만 소통한다

객체는 메시지를 통해 다른 객체의 인터페이스를 호출한다

---

## 공용 인터페이스

외부에 공개된 메서드 집합

- 외부 객체는 이것만 접근 가능
- 내부 구현은 숨겨야 한다

---

## 캡슐화

객체의 내부 상태와 구현을 외부로부터 숨기는 것

- 상태 보호
- 변경 영향 최소화

---

## 인터페이스 vs 구현 분리

- 인터페이스: 무엇을 하는가
- 구현: 어떻게 하는가

이 둘은 분리되어야 한다

---

## 좋은 인터페이스 설계 기준

- 추상적으로 설계하라
- 꼭 필요한 것만 노출하라
- 내부 상태를 직접 노출하지 마라

---

## 요약

객체는 인터페이스를 통해서만 협력하며, 구현은 반드시 내부에 숨겨야 한다

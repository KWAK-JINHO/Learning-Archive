# 추상화와 타입

## 추상화란 무엇인가

복잡한 시스템에서 핵심적인 부분만 남기고 불필요한 세부사항을 제거하는 것

목적

- 복잡성 감소
- 이해 용이성 증가

---

## 객체의 타입은 상태가 아니라 행동으로 결정된다

같은 메시지를 받을 수 있다면 같은 타입이다

예시

- Dog, Cat → speak() 가능 → 같은 타입

---

## 일반화 / 특수화

### 일반화

공통된 특성을 추출하여 상위 개념으로 만드는 것

### 특수화

상위 개념을 구체화하는 것

---

## 분류

객체를 어떤 타입으로 묶을지 결정하는 과정.  
설계 품질에 직접적인 영향을 미친다.

---

## 정적 모델 vs 동적 모델

### 동적 모델

실행 시점의 상태 변화

### 정적 모델

시간과 무관한 구조 (클래스)

---

## 요약

타입은 상태가 아니라 행동을 기준으로 정의되며, 추상화는 복잡성을 줄이기 위한 핵심 도구이다
